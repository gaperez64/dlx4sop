# Benchmark Manifests

This directory contains pinned, self-contained snapshots of the benchmark
corpora used to produce `scoreboard.md`.

## Files

### `corpus.lock.json`

Machine-readable provenance record. For each external source it records:

- `kind`: `committed` (in-tree), `git` (cloned at a pinned commit), or
  `python-package` (generated by the installed package version).
- `url` and `commit` / `version`: exact upstream pin.
- `build`: the exact command(s) that were used to build this source's cases.

The `tiers` array lists the five imported-variable ranges and their manifest
filenames. The `257-512 sample` tier is a source-balanced stratified sample;
`sample_cap_per_source` and `sample_seed` are recorded there.

`tool_versions` records the external tool versions that were active when the
manifests were last regenerated.

### `manifests/dlx4sop-tier-*-manifest.json`

Five per-tier JSON arrays.  Each element follows the same schema as
`tests/qasm_solver_corpus.json` plus provenance fields emitted by
`build_external_qasm_manifest.py`:

| Field | Description |
| --- | --- |
| `name` | Unique case identifier |
| `source` | Human-readable corpus name |
| `source_url` | Upstream URL |
| `source_relative_path` | Path within the upstream repo |
| `qasm_lines` | Inline QASM (self-contained, no network needed) |
| `boundaries` | List of `[input_bits, output_bits]` pairs |
| `max_imported_nvars` | Maximum SOP variable count across all boundaries |
| `max_imported_edges` | Maximum SOP edge count across all boundaries |

The QASM is inlined so benchmarks run fully offline after cloning this repo.

## Refreshing

To rebuild the manifests from scratch (clones FeynmanDD and PyZX, generates
MQT Bench circuits from the installed package):

```sh
python3 tools/build_corpus.py \
    --lock benchmarks/corpus.lock.json \
    --out benchmarks/manifests \
    --work-dir /tmp/dlx4sop-corpus
```

This takes several minutes.  The clones are cached in `--work-dir`; subsequent
runs only re-fetch new commits.

To verify that committed manifests respect their tier variable ranges (fast,
no network if clones already exist):

```sh
python3 tools/build_corpus.py --verify
```

Pass `--rebuild` too to also rebuild and compare case counts (MQT drift is
reported but not treated as a failure since MQT Bench is version-dependent):

```sh
python3 tools/build_corpus.py --verify --rebuild
```

## Updating the pin

1. Run `git ls-remote <url> HEAD` to get the new commit SHA.
2. Edit `corpus.lock.json`: update `commit` and bump `tool_versions` as needed.
3. Re-run `build_corpus.py` and commit the updated manifests alongside the
   updated lock.
