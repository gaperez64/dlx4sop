# MQT Bench manifests

These JSON files list circuits harvested from [MQT Bench](https://github.com/munich-quantum-toolkit/bench)
for use as native-comparison benchmarks against QSOP.

## Files

| File | Description |
| --- | --- |
| `tier-0-32.json` | 0-qubit to 32-qubit circuits (currently empty — MQT Bench circuits in this range overlap with the pre-existing internal corpus and are not separately harvested) |
| `tier-33-64.json` | 34–64 qubit GHZ and BV circuits (36 circuits, qubits 34–64) |
| `tier-65-128.json` | 65–128 qubit GHZ and BV circuits (21 circuits, qubits 66–130) |
| `tier-129-256.json` | 129–256 qubit circuits (currently empty) |
| `tier-257-512-sample.json` | 257–512 qubit sample (currently empty) |
| `harvest-summary.json` | Metadata about the last harvest run (seed, target tiers, size range, per-tier counts) |
| `unsupported.jsonl` | Circuits found during harvest but not supported by the QSOP pipeline (e.g. unsupported gate sets) |

## Circuit families

Only **GHZ** (Greenberger–Horne–Zeilinger) and **BV** (Bernstein–Vazirani) families are present.
Both are Clifford circuits, which means:
- Statevector simulation requires O(2ⁿ) memory — infeasible above ~33 qubits (~272 GB at 34q).
- `qiskit-clifford` stabilizer simulation handles them in O(n²) memory and is the native comparison baseline for the 33-64 and 65-128 tiers.

## Per-entry fields

Each JSON entry in a tier file has:

| Field | Type | Description |
| --- | --- | --- |
| `source` | string | Always `"MQT Bench"` |
| `source_url` | string | MQT Bench GitHub URL |
| `mqt_family` | string | Circuit family (`"ghz"` or `"bv"`) |
| `mqt_algorithm` | string | Algorithm name (same as family for these circuits) |
| `mqt_qubits` | int | Number of qubits |
| `mqt_optimization_level` | int | Qiskit transpilation optimization level (always 1) |
| `qasm_sha256` | string | SHA-256 of the original QASM file |
| `qasm_lines` | list[string] | Full QASM circuit |

## Regenerating

Manifests are regenerated by `scripts/harvest_mqt_bench.py`.
Run it with `--help` for options.
The harvest requires `qiskit`, `mqt-bench`, and `qiskit-aer` installed.
